# 预提取首批领域选择 — 三方开放研究

## 背景

Doramagic 的产品交付模式是：通过 API 提供服务，后端已经对特定领域的核心开源项目完成了灵魂提取和领域共识图谱构建。用户请求到达时，Doramagic 已经"对这个领域有组织过的记忆"，而不是从零开始提取。

这意味着**预提取是一项投资决策**——选错领域 = 浪费提取成本 + 冷启动失败。

### 已验证的技术基础

- 灵魂提取引擎（Soul Extractor v0.9 + P0 补丁）已验证，跨领域适应性好
- 领域共识图谱（Domain Consensus Map）三方研究已完成，架构已定
- 图谱注入提取管线 Stage 0/1 已验证 +20% 提取质量提升（Brick 效应）
- 跨项目智能（公约数 / 打架 / 缝合）已验证
- 预提取技术实现：冷启动自动化管线 7 步（discover → clone → batch_extract → build_fingerprints → compare → build_domain_map → compile_outputs）
- 每个领域预提取 5-7 个精选项目即可形成有效 Alpha 图谱
- 成本估算：7 项目 ≈ $10-40、1-2 小时（并行提取）

### 产品约束

- Doramagic 运行在 OpenClaw 平台上
- OpenClaw 核心能力：cron 定时任务、通知推送、~/clawd/ 文件 I/O、exec 工具调用
- ClawHub 是 OpenClaw 的技能市场（13k+ skills，5.4k+ 已分类）
- 用户群体以开发者为主，但扩展到个人效率/生活场景

## 核心问题

**Doramagic 首批应该预提取哪 5 个领域？**

这 5 个领域将成为 Doramagic 上线时的"出厂知识库"——API 背后的知识库存。

## 选择维度（供参考，不限于此）

### 1. OpenClaw 用户需求信号
- ClawHub 各类别技能数量和下载/安装趋势
- OpenClaw 社区讨论中的高频需求
- "模糊烦恼"密度——用户说不清楚具体要什么，但知道自己有这方面的需求（这正是 Doramagic 最擅长的场景）

### 2. 开源"作业"供应量
- GitHub 对应领域的项目数量和质量（stars、活跃度、多样性）
- 是否有足够多的独立项目（非 fork/套壳）来形成有效的领域共识
- 项目技术栈是否适合灵魂提取（代码结构清晰、有设计决策可提取）

### 3. WHY/UNSAID 提取价值
- 代码 vs 社区知识的差距有多大？（差距越大，Doramagic 价值越高）
- 领域内是否有丰富的"设计哲学"和"踩坑经验"？
- 是否存在"代码说的和实际原因完全不同"的情况？

### 4. OpenClaw 平台适配度
- 该领域的 skill 是否天然匹配 OpenClaw 能力（cron/通知/文件/exec）？
- 平台适配是否需要额外工作？

### 5. 知识通胀/套壳风险
- 领域内是否大量存在"换皮"/"套壳"项目？
- 知识是否已经被过度普及（AI 生成的教程泛滥 → 二阶通胀）？
- Doramagic 的提取能否产生真正的增量价值？

### 6. 冷启动效率
- 5-7 个精选项目能否覆盖领域核心知识？
- 领域是否足够聚焦（不至于太散而无法形成共识）？
- 第一个用户上来时能否立刻感受到价值？

### 7. 战略卡位
- 是否是 Doramagic 能建立先发优势的领域？
- 是否能自然扩展到相邻领域？
- 是否有利于飞轮启动（用户自然贡献新项目）？
