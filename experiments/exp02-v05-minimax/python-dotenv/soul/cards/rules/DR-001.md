title: "Hash (#) in unquoted values is treated as inline comment, silently truncating the value"

rule: |
  If a value contains a # character and is not quoted, python-dotenv interprets
  everything after the # as a comment. The value is silently truncated.
context: |
  The parser regex matches whitespace followed by # as a comment. This follows
  Bash semantics but surprises users who paste URLs or generated passwords.
do:
  - "Always quote values that may contain # characters"
  - "Use double quotes for URLs, color codes, and generated passwords"
dont:
  - "Paste URLs with fragments without quoting"
  - "Use unquoted hex color codes like COLOR=#FF0000"

## 真实场景
你的 .env 文件里有一行 `DATABASE_URL=postgres://user:pass@host/db#pool=5`。
你以为 pool 参数会生效，但 python-dotenv 把 `#pool=5` 当成注释丢掉了。
你的数据库连接没有连接池，在生产环境下性能骤降，排查了两小时才发现。

## 影响范围
- 谁会遇到：所有在 .env 中存储 URL 的开发者
- 什么时候：配置包含 # 字符时
- 多严重：值被静默截断，没有任何报错，极难排查
