  With override=True: .env file > environment > default > empty

  This matters when using ${VAR:-default} syntax and both .env and environment

## 真实场景
你设置了环境变量 `DOMAIN=prod.example.org`，.env 文件里有 `DOMAIN=${DOMAIN:-dev.example.org}`。
使用 override=False (默认值)，期望的 dev.example.org 没有生效，反而用了生产环境的值。
你的开发环境意外连接到了生产数据库。

## 影响范围
- 谁会遇到：同时使用 shell 环境和 .env 文件的开发者
- 什么时候：变量名冲突时
- 多严重：开发环境使用生产配置可能导致数据泄露
