  Setting PYTHON_DOTENV_DISABLED=1 (or any truthy value) completely disables
  load_dotenv() and find_dotenv() from loading .env files.

  This is useful for production, but can cause confusion when accidentally set

## 真实场景
你部署到生产环境，设置了 PYTHON_DOTENV_DISABLED=1 防止加载 .env。
后来同事接手开发，在本地调试时也继承了这个环境变量，
一直奇怪为什么 .env 配置不生效，排查了一下午。

## 影响范围
- 谁会遇到：所有使用 python-dotenv 的开发者
- 什么时候：环境变量意外设置时
- 多严重：配置加载静默失败，难以排查
