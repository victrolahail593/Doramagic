  Since v0.16.0, encoding defaults to 'utf-8' instead of None (system default).

  Previously, the encoding depended on the system's default encoding, which
  could vary between systems (e.g., macOS vs Windows).
do:
  - "Explicitly specify encoding if you need non-UTF-8"
  - "Use encoding='latin-1' for older .env files"
dont:
  - "Assume system default encoding works cross-platform"

## 真实场景
你的 .env 文件包含中文字符，在 macOS 上开发正常。但部署到 CI/CD 的 Linux 容器时，
由于系统默认编码不同，中文字符解析失败，导致配置加载报错。

## 影响范围
- 谁会遇到：使用非 ASCII 字符的开发者
- 什么时候：跨平台部署时
- 多严重：低 - 现在默认 UTF-8，这个问题已缓解
