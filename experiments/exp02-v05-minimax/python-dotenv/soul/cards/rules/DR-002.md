---
card_type: decision_rule_card
card_id: DR-002
repo: python-dotenv
type: BEHAVIOR_CHANGE
title: "Quote mode 'auto' only adds quotes when value contains special characters"
severity: MEDIUM
rule: |
  With quote_mode='auto' (default), python-dotenv only adds quotes when the value
  contains spaces, special characters, or looks like a variable reference.
  Pure alphanumeric values are written unquoted.
context: |
  This changed in v0.19.0. Previously, values were always quoted. Now alphanumeric
  values like 'abc123' are written as abc123 (no quotes).
do:
  - "Use quote_mode='always' if you need consistent quoting"
  - "Be aware that auto mode may change file format across upgrades"
dont:
  - "Rely on auto mode for predictable output format"
  - "Assume values will always be quoted"
confidence: 0.95
sources:
  - "CHANGELOG.md: v0.19.0"
  - "src/dotenv/main.py: set_key() quote handling"
---

## 真实场景
你写了一个脚本自动生成 .env 文件，依赖 quote_mode='auto' 的输出格式来做测试断言。
升级到 v0.19.0 后，原本 `KEY="value"` 的输出变成了 `KEY=value`，你的测试开始失败。

## 影响范围
- 谁会遇到：依赖 .env 文件格式的自动化工具
- 什么时候：升级版本时
- 多严重：测试失败，需要修改断言逻辑
