title: "File mode is now preserved instead of reset to 0o600"
severity: LOW
rule: |
  Since v1.2.2, set_key() and unset_key() preserve the original file mode.
  Previously, they would reset the mode to 0o600 for security.
context: |
  This was changed to respect intentional file permissions while maintaining
  security by not following symlinks.
do:
  - "Check file permissions after upgrading if you rely on specific modes"
  - "Manually set mode if you need specific permissions"
dont:
  - "Assume mode will always be reset to 0o600"
confidence: 0.98

  - "CHANGELOG.md: v1.2.2 Breaking Changes"

## 真实场景
你有一个 .env 文件，权限设置为 0o644（组可读）。升级到 v1.2.2 后调用 set_key()，
权限被保留了 0o644。但你原本期望的安全行为 (0o600) 变了，可能有安全影响。

## 影响范围
- 谁会遇到：依赖特定文件权限的项目
- 什么时候：升级到 v1.2.2+ 后
- 多严重：低 - 主要是权限预期改变
