title: "load_dotenv() does NOT override existing environment variables by default"
severity: MEDIUM
rule: |
  The default override=False means existing environment variables take precedence.
  Values in .env file are ignored if the key already exists in os.environ.
context: |
  This is intentional for safety - production environment variables should not be
  overwritten by development .env files.
do:
  - "Use override=True in .env to force override"
  - "Check that CI/CD environment variables aren't being shadowed"
dont:
  - "Assume .env values will always be loaded"
  - "Rely on .env in production without override=True"
confidence: 0.98
sources:
  - "README.md: Getting Started section"
  - "src/dotenv/main.py: load_dotenv() override parameter"

## 真实场景
你在 CI/CD pipeline 里设置了 `API_KEY=production_key`，然后运行测试套件。
测试套件使用 load_dotenv() 加载 .env 文件，但 .env 里的 `API_KEY=test_key` 没有生效，
因为 override=False (默认值)。你的测试连不上真实的 production API。

## 影响范围
- 谁会遇到：在 CI/CD 环境中运行测试的开发者
- 什么时候：环境变量与 .env 文件冲突时
- 多严重：测试使用错误的凭据，可能导致安全事故
