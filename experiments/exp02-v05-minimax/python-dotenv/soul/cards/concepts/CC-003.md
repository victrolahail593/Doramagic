## Identity
Variable interpolation enables .env values to reference other environment variables using POSIX-style ${VAR} or ${VAR:-default} syntax.

## Is / Is Not

| IS | IS NOT |
|----|--------|
| A mechanism for dynamic value resolution | A template engine |
| Supports default values ${VAR:-default} | Supports complex expressions |
| Resolves at load time | A live binding to future env changes |

## Key Components

| Component | Purpose |
|-----------|---------|
| Variable class | Represents ${VAR} in values |
| Literal class | Represents static text parts |
| parse_variables() | Tokenizes value string into Atom objects |
| resolve() | Recursively resolves Variable to actual values |

## Resolution Order (with override=True)
1. Value in .env file
2. Value in environment
3. Default value (if provided)
4. Empty string
