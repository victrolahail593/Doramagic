# python-dotenv — 项目本质

## 1. 这个项目解决什么问题？
开发者需要管理数据库密码、API Key 等配置，但不想把它们硬编码在代码里或每次手动 export。

## 2. 如果没有这个项目，人们会怎么做？
每次启动终端都手动 export 十几个环境变量，或者把密钥直接写在代码里（不安全），或者用复杂的配置管理工具（太重）。

## 3. 它的核心承诺是什么？
让你把配置写在 .env 文件里，代码里一行 load_dotenv() 就能用，不改变你原来用 os.getenv() 的习惯。

## 4. 它用什么方式兑现承诺？
读取 .env 文件，逐行解析 KEY=VALUE，把键值对注入 os.environ。

## 5. 一句话总结
python-dotenv 让开发者可以把配置存在 .env 文件里自动加载，而不用手动 export 或把密钥硬编码到代码中。
