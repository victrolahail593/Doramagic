# Superpowers — 灵魂

## 1. 解决什么问题？
AI 编码代理（如 Claude Code、Cursor、Codex）在执行开发任务时，容易跳过设计、测试、审查等工程实践，直接写代码导致质量低下。

## 2. 没有它会怎样？
代理会像没有经验的初级工程师一样，直接开始写代码——没有设计讨论、没有 TDD、没有代码审查，最终产出难以维护的技术债务。

## 3. 核心承诺
让 AI 代理像资深工程师一样思考和行动——先理解需求 → 设计方案 → 制定计划 → 写测试 → 实现 → 审查。

## 4. 兑现方式
通过一组可组合的"skills"（技能），在开发流程的不同阶段自动触发，引导代理遵循工程最佳实践。

## 5. 一句话总结
Superpowers 让 AI 编码代理可以遵循完整的软件工程工作流（设计→计划→TDD→审查→完成），而不是直接跳到写代码。

## 6. 设计哲学
**"AI 代理应该像训练有素的工程师一样思考，而不是像没有经验的码农。"**

这是 Superpowers 背后最核心的信念。所有设计决策都围绕一个原则：让代理在写代码之前先理解问题、设计方案、制定计划、写测试。质量优先于速度——自动化的工作流确保每一步都扎实。如果代理不经过设计阶段就写代码，就像没有蓝图的施工；不写测试就交付，就像没有质检的产品。Superpowers 不是要取代人类的判断，而是确保代理在需要人类确认的每个节点都停下来请求批准。

## 7. 心智模型
把 Superpowers 想象成给 AI 代理配备的一位"虚拟技术导师"——它不会替代理写代码，但会在每个关键决策点追问："你确定要这样做吗？""测试在哪？""代码审查了吗？"。这位导师不取代代理的思考能力，而是确保代理的思考路径符合软件工程的常识。
