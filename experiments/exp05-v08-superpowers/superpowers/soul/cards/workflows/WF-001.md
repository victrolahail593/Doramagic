## Overview
The complete workflow from understanding what to build to finishing development: brainstorming → planning → implementation → review.

## Steps

| Step | Skill | Description |
|------|-------|-------------|
| 1 | brainstorming | Before writing code, refine ideas through questions, explore alternatives |
| 2 | using-git-worktrees | Create isolated workspace on new branch |
| 3 | writing-plans | Break work into 2-5 minute tasks with exact file paths |
| 4 | subagent-driven-development | Execute each task with two-stage review |
| 5 | finishing-a-development-branch | Verify tests, present merge/PR options |

## Flow Diagram

```
User Request
     ↓
brainstorming (design approval)
     ↓
using-git-worktrees (new branch + clean state)
     ↓
writing-plans (task list)
     ↓
subagent-driven-development (execute tasks)
     ↓
finishing-a-development-branch (cleanup)
```

## Failure Modes

| Failure | Symptom | Recovery |
|---------|---------|----------|
| Skipping brainstorming | Agent jumps straight to code | Revert, start from step 1 |
| Vague plan | Tasks too large (>5 min) | Rewrite with finer granularity |
| Missing review | Code merged without review | Revert commits, re-run review |
| No test baseline | Can't verify GREEN | Reset to clean state, add tests |
