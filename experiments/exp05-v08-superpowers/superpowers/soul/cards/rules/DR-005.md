rule: |
  Each task in the implementation plan must include:
  - Exact file paths (Create: path/to/file.py, Modify: path/to/file.py:123-145)
  - Exact commands to run (pytest tests/path/test.py::test_name -v)
  - Expected output (Expected: FAIL with "function not defined")
do:
  - "List exact files to create/modify"
  - "Include run commands with expected results"
  - "Add test verification steps"
dont:
  - "Vague file references like 'implement the service'"
  - "Skip verification commands"
confidence: 0.97
sources:
  - "skills/writing-plans/SKILL.md: Task Structure"

## 真实场景
计划里写"实现用户服务"，但没有具体文件路径和命令。
子代理不知道改哪个文件、怎么运行测试。

## 影响范围
- 谁会遇到：使用 subagent-driven-development 的开发者
- 什么时候：写 implementation plan 时
- 多严重：模糊的计划导致执行效率低下
