title: "Subagent-driven development requires TWO stage review"
severity: HIGH
rule: |
  Each task executed by a subagent must go through two sequential reviews:
  1. Spec compliance review (does code match spec?)
  2. Code quality review (is code well-written?)
  Both must pass before moving to next task.
do:
  - "Dispatch implementer subagent first"
  - "Then dispatch spec reviewer subagent"
  - "Then dispatch code quality reviewer subagent"
  - "Fix any issues before proceeding"
dont:
  - "Skip spec review"
  - "Skip code quality review"
  - "Do single review only"
confidence: 0.95

## 真实场景
子代理实现了一个功能，你直接让它继续下一个任务。
但代码可能不符合 spec，或者质量很差。

## 影响范围
- 谁会遇到：使用 subagent-driven-development 的开发者
- 什么时候：每个任务完成后
- 多严重：缺少审查导致代码不符合需求或质量低下
