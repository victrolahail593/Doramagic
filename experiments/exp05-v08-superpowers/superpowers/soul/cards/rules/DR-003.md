title: "TDD: NO production code without a failing test first"
severity: CRITICAL
rule: |
  You cannot write any production code unless a test is already written and verified to fail.
  If you wrote code before tests, delete it and start over.
do:
  - "Write failing test first"
  - "Verify test fails correctly"
  - "Write minimal code to pass"
  - "Delete any code written before tests"
dont:
  - "Write implementation code before tests"
  - "Keep 'reference' code while writing tests"
  - "Adapt existing code while writing tests"
confidence: 0.99

## 真实场景
你直接写了实现代码，然后才开始写测试。这违反了 TDD 原则。
必须删除已写的代码，从测试开始重新来。

## 影响范围
- 谁会遇到：所有实现功能的开发者
- 什么时候：每次写代码时
- 多严重：跳过 TDD 导致代码质量低下、难以维护
