title: "Task plans MUST be bite-sized (2-5 minutes each)"
severity: MEDIUM
rule: |
  Each task in an implementation plan must take 2-5 minutes to complete.
  Break down larger tasks into individual steps: write test, run test, implement code, run test, commit.
do:
  - "Break tasks into atomic steps"
  - "Write 'Step 1: Write failing test', 'Step 2: Run test to verify fail'"
  - "Include exact file paths and commands"
dont:
  - "Write multi-hour tasks like 'Implement feature X'"
  - "Combine test + implementation in one step"
confidence: 0.98
sources:
  - "skills/writing-plans/SKILL.md: Bite-Sized Task Granularity"
---

## 真实场景
你写了一个任务"实现用户登录"，没有拆解。结果子代理不知道从哪开始，
花了 2 小时才完成，还漏了测试。

## 影响范围
- 谁会遇到：使用 subagent-driven-development 的开发者
- 什么时候：写 implementation plan 时
- 多严重：任务太大导致执行混乱、遗漏测试
