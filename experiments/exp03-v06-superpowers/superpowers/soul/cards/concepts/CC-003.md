## Identity
Subagent-Driven Development is the execution methodology that dispatches fresh subagents for each task with two-stage review (spec compliance first, then code quality) to ensure high quality without human-in-the-loop delays.

## Is / Is Not

| IS | IS NOT |
|----|--------|
| A parallel task execution system | Sequential single-agent execution |
| A two-stage review process (spec → quality) | A single code review pass |
| Fresh subagent per task (no context pollution) | Reusing the same agent context |
| Session-persistent (no context switch) | A separate parallel session |

## Key Attributes

| Attribute | Purpose |
|-----------|---------|
| Fresh subagent per task | Avoids context pollution from previous tasks |
| Two-stage review | Spec compliance before code quality |
| Implementer prompt | Defines how subagent implements tasks |
| Spec reviewer prompt | Validates code matches design spec |
| Code quality reviewer | Checks patterns, security, performance |

## Boundaries
- Starts at: Implementation plan exists with independent tasks
- Requires: Tasks are mostly independent (not tightly coupled)
- Does NOT: Handle tightly coupled tasks (use executing-plans instead)
