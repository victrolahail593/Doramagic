## Identity
Brainstorming is the mandatory pre-implementation phase that forces AI to explore intent, requirements, and design through collaborative dialogue BEFORE any code is written.

## Is / Is Not

| IS | IS NOT |
|----|--------|
| A structured requirements gathering process | A quick idea validation |
| A multi-phase exploratory dialogue | A single-pass Q&A |
| A design presentation and approval workflow | A implementation planning session |
| Mandatory for ALL projects (even "simple" ones) | Optional for small tasks |

## Key Attributes

| Attribute | Purpose |
|-----------|---------|
| One question at a time | Prevents overwhelming the user |
| Multiple choice preferred | Easier for user to answer |
| 2-3 approach proposals with trade-offs | Ensures design alternatives explored |
| Incremental validation | Gets user approval after each section |
| Design document output | Saves to docs/plans/YYYY-MM-DD-<topic>-design.md |

## Boundaries
- Starts at: User wants to build/create/modify something
- Ends at: User approves design AND writing-plans skill invoked
- Does NOT: Proceed to implementation, scaffold, or write code
