title: "Skills Not Being Invoked -- Installation or Invocation Issues"
severity: HIGH
rule: |
  If users report skills are not being triggered or cannot be invoked,
  it may be due to: 1) Skill not properly installed/symlinked,
  2) Using wrong namespace (e.g., /superpowers:brainstorm vs /brainstorm),
  3) Model rationalizing that skill is not needed.
do:
  - "Verify skill installation: ls -la ~/.agents/skills/superpowers"
  - "Use correct namespace: superpowers:skill-name"
  - "Check SessionStart hook is loading using-superpowers"
  - "Restart the agent after installation"
dont:
  - "Assume skills are working if user doesn't complain"
  - "Ignore rationalization red flags"
  - "Skip skill invocation for 'simple' tasks"
confidence: 0.90

  - "GitHub Issue #58 (23 comments, bug label)"

## 真实场景
用户在 Claude Code 里装了 superpowers 插件，问"帮我计划这个功能"。
你直接开始写代码，没有触发 brainstorming。用户以为 superpowers 坏了，
其实是你跳过了技能检查。后来用户装了 skill 但不知道怎么验证是否生效。

## 影响范围
- 谁会遇到：Superpowers 新用户
- 什么时候：首次安装或遇到问题时
- 多严重：HIGH — 技能不生效 = 整个工作流失效
