title: "Fresh Subagent Per Task -- No Context Pollution"
severity: HIGH
rule: |
  In subagent-driven-development, dispatch a FRESH subagent for EACH task.
  Reusing the same agent context leads to context pollution and degraded quality.
  Each task should start with clean context to ensure focused implementation.
do:
  - "Dispatch new subagent for every task"
  - "Pass full context (spec, plan, files) to each subagent fresh"
  - "Let each subagent complete fully before next one"
dont:
  - "Reuse same agent for multiple tasks"
  - "Assume context carries over correctly"
  - "Skip context passing to 'save time'"
confidence: 0.95

  - "skills/subagent-driven-development/SKILL.md:L1-8"

## 真实场景
你用同一个 subagent 完成了 5 个任务。第二十个任务时，agent 的上下文里
还残留着第一个任务的想法和代码片段。它不小心把第一个任务的函数名
带到了第十个任务里，导致奇怪的 bug。你花了 2 小时排查，发现是上下文污染。

## 影响范围
- 谁会遇到：使用 subagent-driven-development 的用户
- 什么时候：执行多个独立任务时
- 多严重：HIGH — 上下文污染会导致难以追踪的 bug
