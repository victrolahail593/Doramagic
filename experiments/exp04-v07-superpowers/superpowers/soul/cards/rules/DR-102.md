repo: obra/superpowers
type: COMMUNITY_GOTCHA
title: "Installation Path Issues -- Wrong Directory or Missing Symlinks"
severity: HIGH
rule: |
  Superpowers requires correct installation paths that vary by platform:
  - Claude Code: ~/.claude/plugins/ or marketplace
  - Codex: ~/.codex/superpowers + ~/.agents/skills/superpowers symlink
  - OpenCode: ~/.config/opencode/superpowers + plugins + skills symlinks
  Wrong paths or missing symlinks will cause skills to not be discovered.
do:
  - "Follow platform-specific installation exactly"
  - "Create symlinks as documented (rm before ln -s)"
  - "Restart the agent after installation"
  - "Verify: ls -la to check symlinks exist"
dont:
  - "Assume 'git clone' alone is enough"
  - "Skip symlink creation steps"
  - "Use deprecated paths (e.g., ~/.codex/skills/)"
confidence: 0.92
sources:
  - "GitHub Issue #76 (17 comments, bug label)"
  - ".codex/INSTALL.md"
  - ".opencode/INSTALL.md"

## 真实场景
用户在 Codex 上安装 superpowers，git clone 后直接开始问问题。
Skills 没有触发。用户以为 skill 内容有问题，实际上是 skills 目录
根本没有被 agent 发现 — 缺少了创建 symlink 的那一步。

## 影响范围
- 谁会遇到：所有平台的新用户
- 什么时候：首次安装时
- 多严重：HIGH — 安装不正确 = 技能完全不可用
