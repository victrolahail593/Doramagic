不能，而且不建议这样做。

根据 `writing-skills` 文档的说明：

> **Why no @ links:** `@` syntax force-loads files immediately, consuming 200k+ context before you need them.

`@` 语法的作用是**立即将文件内容注入到上下文**，而不是"触发"一个 skill 去执行。两者有本质区别：

| 方式 | 效果 |
|------|------|
| `@skills/tdd/SKILL.md` | 把文件内容硬塞进上下文，浪费 token |
| `Skill` 工具调用 | 按需加载，由 Claude 主动决策执行 |

Skill 的"触发"是由 Claude 读取 `description` 字段后**主动用 `Skill` 工具调用**来实现的，不是被动地通过文件引用。

**正确的跨 skill 引用写法：**

```markdown
# ❌ 错误
@skills/superpowers/test-driven-development/SKILL.md

# ✅ 正确
**REQUIRED SUB-SKILL:** Use superpowers:test-driven-development
```

这样 Claude 读到文档时会知道"此处需要调用 TDD skill"，但不会在不需要时白白消耗 context。
