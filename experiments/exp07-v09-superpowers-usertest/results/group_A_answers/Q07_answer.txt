常见原因和排查思路如下：

## 最常见原因

### 1. SessionStart hook 未配置或未触发
`using-superpowers` 技能依赖 SessionStart hook 在对话开始时自动加载。检查：

```bash
cat ~/.claude/settings.json
```

应该包含类似：
```json
{
  "hooks": {
    "SessionStart": [...]
  }
}
```

如果没有，skills 不会在对话开始时被注入到上下文。

### 2. 技能文件路径不正确
Superpowers 需要技能文件在正确位置。确认：
```bash
ls ~/.claude/skills/superpowers/
# 应该能看到 brainstorming.md, test-driven-development.md 等文件
```

### 3. Agent 是 subagent（最容易被忽略的原因）
注意 `using-superpowers` 技能里有这段：
```
<SUBAGENT-STOP>
If you were dispatched as a subagent to execute a specific task, skip this skill.
</SUBAGENT-STOP>
```

如果你的 agent 是被另一个 agent 调用的子 agent，它会**故意跳过** skills。

### 4. 模型没有调用 Skill tool
即使 hook 触发了，模型需要主动调用 `Skill` 工具。如果模型认为"这只是个简单问题"，它可能跳过。这是 `using-superpowers` 技能里列出的 **Red Flags** 场景。

---

## 排查步骤

1. **确认 hook 是否触发** — 看对话开始有没有 `SessionStart:startup hook success` 的系统消息（就像本对话顶部那样）
2. **确认技能被加载** — 对话开始时系统消息里应该有可用 skills 列表
3. **直接告诉 agent** — 说 "请先用 brainstorming skill"，看它是否响应
4. **检查 Superpowers 版本** — 运行安装脚本的更新命令重新安装

你现在看到的对话（本对话）顶部有 `SessionStart:startup hook success`，说明 hook 是正常的。问题大概率出在**另一个对话/agent 实例**的配置上。能分享一下那个 agent 是怎么启动的吗？
