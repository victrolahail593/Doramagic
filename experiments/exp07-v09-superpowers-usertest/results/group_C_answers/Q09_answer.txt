根据我对 superpowers 项目的了解，**superpowers 并没有 auto-fix 功能**。

你可能混淆了几个概念：

1. **superpowers 不是 linter**：它不是像 ESLint 那样的静态分析工具，不会自动扫描代码并提供修复建议。

2. **superpowers 是工作流框架**：它的核心是一套 skills（技能），用于引导 AI 代理按照结构化的工作流完成任务，比如：
   - `test-driven-development`：强制先写测试再写实现
   - `systematic-debugging`：系统化调试流程
   - `code-review-expert`：代码审查
   - `verification-before-completion`：完成前验证

3. **可能相关的功能**：
   - `simplify` skill：审查已修改的代码，检查复用性、质量和效率，然后修复发现的问题
   - `code-review-expert` skill：用高级工程师视角审查当前 git 变更，检测 SOLID 违规、安全风险等
   - `dimension-explosion-testing` skill：发现 bug 后，强制识别所有可能受影响的系统维度

如果你想要类似 auto-fix 的体验，可以：

```bash
# 让 Claude 使用 simplify skill 审查代码
/simplify

# 或者使用 code-review-expert 审查 git 变更
/code-review-expert
```

但这些都需要手动触发，不是自动后台运行的。superpowers 的设计哲学是"在关键节点强制介入"，而不是"静默自动修复"。
