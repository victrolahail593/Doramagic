根据知识注入文件，这是一个已知的常见问题，有几个可能的原因：

## 可能原因

**1. Skill 静默失效（最常见）**

AI 用的是 Claude Code 原生的 Plan Mode，不是 superpowers 的工作流。你看起来它在"计划"，但其实完全没触发任何 skill。这个问题在 Issue #189 有大量讨论。

**根本原因：** `disable-model-invocation` 配置问题。

**解决方法：**
```
/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace
# 然后重启 Claude Code
```

**2. 安装路径错误**

Claude Code 只在 `~/.claude/plugins/` 里找 skills。如果你把 superpowers clone 到了其他目录，skills 根本不可用，但也不会报错。

**解决方法：** 严格按照对应平台的 `INSTALL.md` 操作，不要自定义路径。

## 验证是否真正激活

安装后，直接问 Claude：

> **"你现在在用哪个 skill？"**

如果它答不上来或者说"没有用任何 skill"，说明 superpowers 没有正确激活。

## 排查顺序

1. 确认安装路径是 `~/.claude/plugins/`
2. 重新执行 marketplace 安装命令
3. 重启 Claude Code
4. 用一个新对话测试，问它是否识别到 brainstorming skill
