  If a skill might apply (even 1% chance), you MUST invoke it.

  - "Always check if a skill applies before responding"
  - "When in doubt, invoke the skill to verify"

  - "Skip skill invocation because you 'already know what to do'"
  - "Rationalize that current approach is faster"

## 真实场景
用户说"帮我修复这个 bug"。你打算直接开始调试，但其实 systematic-debugging skill 应该被触发。你觉得自己能搞定，不需要那个 skill——然后花了 2 小时在错误的方向上。

## 影响范围
- 谁会遇到：所有使用 superpowers 的开发者
- 什么时候：任何时候你想跳过 skill 时
- 多严重：跳过 skill 导致效率低下、方法不对、遗漏关键步骤
