rule: |
  Implementation tasks must be small enough to complete in 2-5 minutes.
  Each task should have exact file paths, complete code, and verification steps.
do:
  - "Break down large tasks into 2-5 minute chunks"
  - "Each task should be independently verifiable"
  - "Tasks should be atomic: one concern per task"
dont:
  - "Create tasks that take 30+ minutes"
  - "Combine multiple concerns in one task"
  - "Write vague task descriptions"

## 真实场景
你写了一个任务"实现用户认证系统"。subagent 收到后花了 2 小时才完成，中途遇到很多问题但你不在旁边指导。最后发现实现偏离了 spec 好几处，需要大幅返工。

## 影响范围
- 谁会遇到：使用 subagent-driven-development 的开发者
- 什么时候：创建 implementation plan 时
- 多严重：任务太大导致 subagent 失控、偏离计划、难以审查
