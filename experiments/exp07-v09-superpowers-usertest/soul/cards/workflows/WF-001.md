## Steps

1. **brainstorming** (skills/brainstorming/SKILL.md)
   - 用户提出需求 → 代理追问澄清 → 产出分块设计文档 → 用户确认

2. **using-git-worktrees** (skills/using-git-worktrees/SKILL.md)
   - 设计批准 → 创建隔离 workspace（新分支 + worktree）→ 验证干净测试基线

3. **writing-plans** (skills/writing-plans/SKILL.md)
   - 批准的设计 → 拆解为 2-5 分钟粒度的任务 → 每个任务有精确文件路径、完整代码、验证步骤

4. **subagent-driven-development** (skills/subagent-driven-development/SKILL.md)
   - 计划就绪 → 每个任务分发 fresh subagent → 两阶段审查 → 继续下一个任务

5. **test-driven-development** (skills/test-driven-development/SKILL.md)
   - 实现每个任务时遵循 Red-Green-Refactor

6. **requesting-code-review** (skills/requesting-code-review/SKILL.md)
   - 任务间 → 审查与计划的一致性 → 按严重程度报告

7. **finishing-a-development-branch** (skills/finishing-a-development-branch/SKILL.md)
   - 任务完成 → 验证测试 → 提供选项（合并/PR/保留/丢弃）→ 清理 worktree

## Flow Diagram

```dot
digraph workflow {
    rankdir=TB;
    User -> brainstorming;
    brainstorming -> "design approved?";
    "design approved?" -> using-git-worktrees [label="yes"];
    "design approved?" -> brainstorming [label="no"];
    using-git-worktrees -> writing-plans;
    writing-plans -> subagent-driven-development;
    subagent-driven-development -> requesting-code-review;
    requesting-code-review -> "all tasks done?";
    "all tasks done?" -> finishing-a-development-branch [label="yes"];
    "all tasks done?" -> subagent-driven-development [label="no"];
}
```

## Failure Modes

| Step | Failure | Recovery |
|------|---------|----------|
| brainstorming | 用户不确认 design | 继续迭代，直到确认 |
| writing-plans | 任务粒度太大 | 重新拆解为 2-5 分钟 |
| subagent-driven-development | subagent 无法完成任务 | 手动介入或重新规划 |
| test-driven-development | 写测试前写了生产代码 | 删除代码，从测试重新开始 |
