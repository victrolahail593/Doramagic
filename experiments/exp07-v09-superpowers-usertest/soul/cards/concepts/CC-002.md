## Identity
Subagent-Driven Development 是 superpowers 的核心执行模型：通过 fresh subagent 执行每个任务，配合两阶段审查（先 spec 合规性，后代码质量），实现高质量快速迭代。

## Is / Is Not

| IS | IS NOT |
|----|--------|
| 同一会话内的任务分发 | 需要新建会话的并行执行 |
| Fresh subagent 避免上下文污染 | 单一 agent 连续执行多任务 |
| Spec 审查 + Code Quality 审查双保险 | 仅做语法检查 |

## Boundaries
- 开始：读取 plan，提取所有任务，创建 TodoWrite
- 每任务：dispatch implementer → implement → spec review → code quality review → complete
- 不处理：任务间的强耦合依赖（需要手动协调）
