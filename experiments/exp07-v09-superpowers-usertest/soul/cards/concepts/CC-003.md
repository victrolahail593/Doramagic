## Identity
Red-Green-Refactor 是 superpowers 强制执行的测试驱动开发循环：先写失败的测试（Red），再写最小代码让它通过（Green），最后重构（Refactor）。任何在测试之前写的生产代码必须删除。

## Is / Is Not

| IS | IS NOT |
|----|--------|
| 铁律：测试优先 | 可选的测试覆盖 |
| 先失败再通过 | 先写代码后补测试 |
| 写测试前的代码必须删除 | "保留参考" |

## Key Attributes

| Attribute | Type | Purpose |
|-----------|------|---------|
| RED | phase | 写一个最小测试，展示预期行为 |
| GREEN | phase | 写最小代码让测试通过 |
| REFACTOR | phase | 重构代码，保持测试绿色 |

## Boundaries
- 开始：任何新功能、Bug 修复、重构
- 例外：一次性原型、生成代码、配置文件（需人类确认）
- 不处理：手动测试探索
