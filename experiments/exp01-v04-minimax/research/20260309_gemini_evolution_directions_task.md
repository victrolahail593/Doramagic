# Gemini 研究任务：Soul Extractor 三个进化方向的产品价值分析

## 背景

Soul Extractor 是一个从 GitHub 开源项目自动提取结构化知识的工具，产出 .cursorrules / CLAUDE.md 作为"AI 燃料"。v0.6 已验证通过，包含社区信号采集。

### 一个关键实验的发现

我们用 Soul Extractor v0.6 对 Superpowers（一个 AI 技能框架，github.com/obra/superpowers）进行了提取。同时，我们的研究员（Claude Opus）也对 Superpowers 做了独立的深度研究分析。

**两种分析的产出差异巨大：**

| 维度 | 人类研究（Opus 深度分析） | v0.6 自动提取 |
|------|------------------------|--------------|
| 层次 | 战略 + 哲学 + 设计意图 | 操作 + 结构 + 规则 |
| 代表产出 | "反合理化设计源于说服力研究，对 LLM 同样有效" | "DR-001: 1% Rule — 有可能就必须调用" |
| 价值 | 帮建设者做设计决策 | 帮 AI 避开操作陷阱 |
| 知识层 | WHY（为什么这样设计） | WHAT + HOW + IF |

这个实验揭示了 Soul Extractor 的三个重大进化方向。

## Opus 对 Superpowers 的战略洞察（完整版）

以下是我们研究员的深度分析——这些 WHY 层知识是 v0.6 自动提取完全没有捕获的：

1. **互补关系**：Superpowers = 方法论注入（教 AI 怎么干活），Soul Extractor = 知识注入（教 AI 懂什么库）。一个改行为，一个补知识。产出格式应兼容。

2. **5 个借鉴的设计模式**：
   - 反合理化设计（源于 Cialdini 说服力研究，对 LLM 有效）
   - Description 是触发器不是摘要（防止模型跳过完整指令）
   - 技能链（每个 skill 知道下一步调谁）
   - 上下文是公共资源（严控 token 量）
   - Iron Law / Hard Gate（零例外语言）

3. **战略启示**：Superpowers 是人手写的方法论框架，Soul Extractor 是自动化知识提取管道。差异化在"自动化"。

## 三个进化方向

### 方向 1：验证闭环（P0）
当前产出没有验证。规则卡的 sources 是否真实？场景是否编造？severity 是否合理？全靠信任模型。

### 方向 2：WHY 层提取（P1）
从"文档级认知"到"专家级认知"的跃迁。WHY 层知识是专家和新手的核心差距。

### 方向 3：拆解/定向注入（P2）
一个重要的认知纠正——拆解不应该按"代码模块"拆，应该按"问题域"拆。因为 AI 是为解决问题服务的，不是为代码而代码。

以 python-dotenv 为例：
- 代码模块拆解：main.py / parser.py / variables.py / cli.py
- **问题域拆解**：配置加载 / 配置修改 / 配置发现 / CLI 工具

用户关心的是"我要解决配置加载问题"，不是"我要用 main.py"。
