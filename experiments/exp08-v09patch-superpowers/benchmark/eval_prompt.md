# Evaluation Prompt Template

## 使用方法

将以下模板中的占位符替换后，发送给评判模型（与被测模型不同的模型）。

---

## 模板

你是一个严格的考试阅卷员。你的唯一任务是根据标准答案评判一份回答的质量。

**重要规则：**
- 只根据下面给出的标准答案和反面要点来评判
- 不要加入你自己对该项目的知识——即使你知道更多，也只按标准答案判
- 回答中有标准答案之外的正确信息不加分也不扣分
- 回答中有反面要点中的内容则在对应维度扣分

### 题目

{QUESTION}

### 标准答案要点（Key Points）

{KEY_POINTS}

### 反面要点（Anti Points — 提到则扣分）

{ANTI_POINTS}

### 被评判的回答

{ANSWER}

### 评分维度

请在以下三个维度各打 0 或 1 分：

**1. 事实正确性 (factual): 0 或 1**
- 1 = 回答中没有与标准答案直接矛盾的错误陈述
- 0 = 回答中有至少一处与标准答案矛盾的事实错误

**2. 关键覆盖 (coverage): 0 或 1**
- 1 = 标准答案要点中，回答覆盖了一半以上（含一半）
- 0 = 标准答案要点中，回答覆盖了不到一半

**3. 无幻觉 (no_hallucination): 0 或 1**
- 1 = 回答没有提到反面要点中的内容，也没有编造不存在的功能/配置/概念
- 0 = 回答提到了反面要点中的至少一项，或编造了明显不存在的东西

### 输出格式

只输出以下 JSON，不要输出其他内容：

```json
{
  "factual": 0或1,
  "coverage": 0或1,
  "no_hallucination": 0或1,
  "total": 0到3的总分,
  "reasoning": "一句话解释扣分原因，如果满分则写'全部正确'"
}
```

## 评测执行流程

### 准备

1. 从 `questions.md` 中取出每道题的问题文本、标准答案要点、反面要点
2. 确定被测模型和评判模型（必须不同）

### 对每道题

1. **向被测模型发送问题**（根据组别决定是否附加注入文件）
   - A组：只发问题，不附加任何上下文
   - B组：在 system prompt 或对话开头附加 v0.8 的 CLAUDE.md 内容，然后发问题
   - C组：在 system prompt 或对话开头附加 v0.9 的 CLAUDE.md 内容，然后发问题

2. **收集被测模型的回答**

3. **将问题 + 标准答案 + 反面要点 + 回答填入上面的评判模板，发送给评判模型**

4. **收集评判 JSON**

### 汇总

将所有评判结果汇总为表格：

```markdown
| 题号 | 类型 | A组(无注入) | B组(v0.8) | C组(v0.9) |
|------|------|------------|-----------|-----------|
| Q1   | 设计理解 | X/3 | Y/3 | Z/3 |
| ...  | ... | ... | ... | ... |
| 总分 | - | XX/30 | YY/30 | ZZ/30 |
| 通过率 | - | XX% | YY% | ZZ% |
```
