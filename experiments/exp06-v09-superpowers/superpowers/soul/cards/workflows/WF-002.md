## Overview
The strict three-phase cycle required for all production code: write failing test → minimal code to pass → refactor.

## Steps

| Phase | Action | Goal |
|-------|--------|------|
| RED | Write minimal failing test | Define expected behavior |
| GREEN | Write minimal code to pass | Make test pass, no more |
| REFACTOR | Clean code while keeping tests green | Improve design |

```
    └─────────────────────────────────────┘
                    ↓ Yes
    ┌─────────────────────────────────────┐
    │  REFACTOR: Clean up                 │
    └─────────────────────────────────────┘
                    ↓
    ┌─────────────────────────────────────┐
    │  Next Task                          │
    └─────────────────────────────────────┘
```

## Failure Modes

| Failure | Symptom | Recovery |
|---------|---------|----------|
| Writing code before test | "I already wrote the code" | Delete it, start over |
| Not watching test fail | Test passes on first run | Delete, write proper test |
| Writing too much code | GREEN exceeds what's needed | Delete, write minimal |
| Refactor breaks tests | Tests fail after refactor | Revert refactor |
