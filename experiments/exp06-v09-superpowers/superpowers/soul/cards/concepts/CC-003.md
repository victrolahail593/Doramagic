## Identity
A development pattern where each implementation task is executed by a fresh subagent, followed by a two-stage review (spec compliance first, then code quality) before moving to the next task.

## Is / Is Not

| IS | IS NOT |
|----|--------|
| Parallel task execution within same session | Sequential manual implementation |
| Fresh subagent per task (no context pollution) | Single agent doing everything |
| Two-stage review: spec → quality | Single review pass |
| Fast iteration (no human-in-loop between tasks) | Waiting for human approval at each step |

## Key Attributes

| Attribute | Type | Purpose |
|-----------|------|---------|
| implementer_subagent | role | Executes the task, writes code, tests, commits |
| spec_reviewer | role | Verifies code matches the spec |
| code_quality_reviewer | role | Checks code quality (style, patterns, etc.) |
| two_stage_gates | array | [spec_approved, quality_approved] |

## Boundaries
- Starts at: When implementation plan exists and tasks are mostly independent
- Requires: Clear task definitions with file paths
- Does NOT handle: Tightly coupled tasks (use executing-plans instead)
