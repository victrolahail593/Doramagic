根据 writing-plans skill 的定义，**bite-sized = 每步只做一个动作，2-5 分钟内完成**。

---

## 差的例子（不是 bite-sized）

```markdown
### Task 1: 实现用户登录功能

- 写登录接口
- 加验证逻辑
- 写测试
- 提交
```

问题：含糊、多合一、没有明确的输入/输出/命令。代理不知道先写什么，很容易跳过测试或漏掉细节。

---

## 好的例子（bite-sized）

```markdown
### Task 1: 实现登录验证

**Step 1: 写失败的测试**
```python
def test_login_invalid_password():
    result = login("user@example.com", "wrong")
    assert result is None
```

**Step 2: 运行确认测试失败**
Run: `pytest tests/test_auth.py::test_login_invalid_password -v`
Expected: FAIL with "login not defined"

**Step 3: 写最小实现**
```python
def login(email, password):
    user = db.find(email)
    return user if user and user.password == hash(password) else None
```

**Step 4: 运行确认通过**
Run: `pytest tests/test_auth.py::test_login_invalid_password -v`
Expected: PASS

**Step 5: 提交**
```bash
git add src/auth.py tests/test_auth.py
git commit -m "feat: add login password validation"
```
```

---

## 核心区别

| | 差的例子 | 好的例子 |
|---|---|---|
| 步骤粒度 | 多个动作合并 | 每步一个动作 |
| 可验证性 | 无法确认是否完成 | 有明确命令和预期输出 |
| TDD | 测试是事后补 | 测试先于实现 |
| 执行时间 | 未知 | 每步 2-5 分钟 |

本质上：**bite-sized = 可以交给另一个人（或子 agent）独立执行的最小单元，执行完立即能验证对错。**
