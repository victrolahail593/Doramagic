根据知识注入文件和 Superpowers 的设计，agent 忽略 skills 通常有以下几个原因：

## 可能的问题

### 1. Marketplace 未配置（最常见）
安装顺序错误。必须先配置 marketplace，再安装 plugin：
```
/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace
```
如果跳过第一步，skills 根本不存在，agent 自然无法触发。

### 2. Skills 调用机制混淆
不要用 `Read` tool 直接读 skill 文件——那是两套不同的加载机制。必须通过 `Skill` tool 调用，或者用 `/superpowers:brainstorming` 等斜杠命令触发。

### 3. `disable-model-invocation` 限制
部分 skill 在 Claude Code 中会遇到这个错误（Issue #189），这是平台限制，不是配置问题。如果特定 skill 报这个错，需要等待维护者修复。

### 4. 没有可见反馈导致误判
知识包明确指出（Issue #446 仍 open）：Superpowers **没有明显的视觉反馈**告诉你"skill 已触发"。你以为 agent 在忽略 skills，实际上它可能已经在执行，只是你看不出来。

## 排查步骤

1. **验证安装**：手动执行 `/superpowers:brainstorming`，看是否有响应
2. **检查 marketplace 配置**：确认两步安装都完成了
3. **测试触发条件**：直接说"我要新增一个功能"，看 agent 是否主动调用 brainstorming
4. **确认调用方式**：只用 `Skill` tool 或斜杠命令，不要用 `Read` tool 读 skill 文件

## 根本原因

最可能的情况是安装步骤不完整（marketplace 未配置），skills 根本没有注册进系统，所以 agent 找不到、也就不会触发。
