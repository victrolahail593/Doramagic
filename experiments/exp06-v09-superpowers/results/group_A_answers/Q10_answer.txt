## `@` 语法的行为

`@` 语法**确实有效**——它会强制立即加载被引用的文件。但 writing-skills 里明确**不推荐**这样做：

> **Why no @ links:** `@` syntax force-loads files immediately, consuming 200k+ context before you need them.

## 正确的跨 skill 引用方式

写设计文档时，应该用文本引用而非 `@`：

```markdown
# ❌ Bad
@skills/test-driven-development/SKILL.md

# ✅ Good  
**REQUIRED SUB-SKILL:** Use superpowers:test-driven-development
```

## 实际效果的区别

| 方式 | 效果 |
|------|------|
| `@test-driven-development` | 立即把整个文件内容塞进 context，无论当前是否需要 |
| 文本引用 `REQUIRED SUB-SKILL: ...` | Claude 读到这行时主动去 invoke Skill tool，按需加载 |

## 总结

在 brainstorming 文档里写 `@test-driven-development` 技术上会触发加载，但这会提前消耗大量 context。推荐做法是写 `**REQUIRED SUB-SKILL:** Use superpowers:test-driven-development`，这样 Claude 执行到该步骤时才会按需调用，不会提前浪费 context。
