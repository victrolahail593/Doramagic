根据知识注入文件，这个问题有几个常见原因：

## 可能原因

### 1. Plugin 未真正激活（最常见）
对应 Issue #58（23 comments）：marketplace 配置不正确，plugin 看起来装上了，但实际上没有激活。Skills 没有安装成功的视觉确认，你完全靠猜。

**排查方法：**
- 检查 Claude Code plugin marketplace，确认 Superpowers 状态是"已激活"而不只是"已安装"
- 尝试卸载后重新安装，重启 Claude Code

### 2. `disable-model-invocation` 限制
对应 Issue #189（22 comments）：某些 skill（如 brainstorming）配置了 `disable-model-invocation`，导致 Skill tool 调用直接报错或静默失败。这是 Claude Code 本身对特定 skill 配置的限制，不是用户操作错误。

**排查方法：**
- 手动尝试调用一个 skill，看是否有报错信息
- 如果有 `disable-model-invocation` 错误，需要等维护者修复，暂时无法绕过

### 3. Skills 工作但没有视觉反馈
对应 Issue #446（20 comments，至今 open）：skills 可能在工作，但你感知不到。

**判断方法：**
- 观察 agent 是否在响应开头说了 `"Using [skill] to [purpose]"` 这样的声明
- 观察是否用 TodoWrite 创建了任务列表
- 如果这两个都没有，skills 确实没有触发

## 解决步骤

1. **先验证安装**：重新检查 marketplace，确保 plugin 真正激活，不是仅安装
2. **测试 skill 调用**：手动输入 `/superpowers:brainstorming` 或通过 Skill tool 调用，看有无错误
3. **检查 session start**：skills 的触发依赖 `using-superpowers` skill 在对话开始时被加载——如果 session start hook 没有正确执行，整个触发链就断了
4. **如果是 `disable-model-invocation` 错误**：报告给维护者，等待修复

## 根本原因总结

> Skills 触发最自然的失败模式是"静默失败"——没有报错，没有提示，agent 就像什么都没安装一样直接写代码。这正是这个问题难以排查的原因。
