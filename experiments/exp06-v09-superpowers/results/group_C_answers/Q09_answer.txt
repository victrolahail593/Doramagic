根据注入的知识文件，Superpowers **没有** "auto-fix" 功能。

知识包中完整记录了 Superpowers 的核心能力：

- Skills 工作流（brainstorming、TDD、debugging、code review 等）
- 子代理驱动开发（subagent-driven development）
- 计划编写与执行
- 两阶段代码审查

文档中没有任何关于类似 ESLint `--fix` 的自动代码扫描或修复功能的描述。

这个功能很可能不存在，或者是对其他功能的误解。如果你的目标是：

- **代码质量检查** → 使用 `superpowers:requesting-code-review` skill
- **Bug 修复** → 使用 `superpowers:systematic-debugging` skill
- **代码简化** → 使用 `simplify` skill

建议不要基于未经文档证实的功能调整工作流。
