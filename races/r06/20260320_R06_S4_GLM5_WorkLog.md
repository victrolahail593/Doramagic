### Step 1: Stage 1.5 Agentic 实现

**输出目录**: `races/r06/agentic/s4-glm5/`

#### 1.1 tools.py (~500行)

实现 5 个工具：

| 工具 | 用途 | 输入 | 输出 |
|------|------|------|------|
| `read_artifact` | 读取 Stage 0/1 产出 | artifact_name | 文件内容 |
| `list_tree` | 列出目录结构 | path | 目录树 |
| `search_repo` | 搜索代码内容 | query, paths | 匹配结果 |
| `read_file` | 读取文件内容 | path, start_line, end_line | 文件片段 |
| `append_finding` | 记录发现 | claim | 确认记录 |

关键类：
- `AgentContext` — Agent 上下文（repo 路径、findings、已读文件等）
- `ToolExecutor` — 工具执行器
- `ToolResult` — 工具执行结果

#### 1.2 prompts.py (~300行)

LLM prompt 模板：

| 函数 | 功能 |
|------|------|
| `build_agent_prompt()` | 构建 Agent prompt |
| `build_context_digest_prompt()` | 构建上下文摘要 prompt |
| `parse_llm_response()` | 解析 LLM 响应 |

Prompt 设计要点：
- 系统提示定义 Agent 角色和能力
- 每轮 fresh context，不依赖长对话记忆
- 明确告诉 Agent 可用工具和预算

#### 1.3 agent_loop.py (~585行)

核心类：

```python
@dataclass
class AgentState:
    current_round: int = 0
    current_hypothesis_index: int = 0
    tool_calls_used: int = 0
    prompt_tokens_used: int = 0
    consecutive_no_gain: int = 0
    resolved_hypotheses: list[str]
    exploration_log: list[ExplorationLogEntry]
    claim_ledger: list[ClaimRecord]
    ...

class AgentLoop:
    def __init__(self, input_data, llm_adapter): ...
    async def run(self) -> ModuleResultEnvelope[Stage15AgenticOutput]: ...
    def _should_continue(self) -> bool: ...
    def _check_budget(self) -> bool: ...
    async def _process_round(self, hypothesis) -> bool: ...
```

关键逻辑：
1. 按优先级排序 hypotheses
2. 每轮聚焦一个 hypothesis
3. LLM 决定调用哪个工具
4. 执行工具，更新状态
5. 检查终止条件（budget/无进展/全部解决）

### Step 2: Domain Bricks 积木制作

**输出目录**: `races/r06/bricks/s4-glm5/`

#### 2.1 brick_schema.py (~280行)

积木 Schema 定义：

| 类 | 功能 |
|---|------|
| `BrickLayer` | 层级枚举 (L1/L2) |
| `BrickSourceFrontmatter` | YAML frontmatter |
| `BrickContent` | 积木内容 |
| `BrickSourceFile` | 源文件完整结构 |
| `BrickCompilationResult` | 编译结果 |
| `estimate_tokens()` | Token 估算 |

双层架构：
- **L1 框架级**: 200-400 tokens, WHY 哲学 + 心智模型
- **L2 模式级**: 400-800 tokens, UNSAID + 速查规则 + 反模式

#### 2.2 brick_compiler.py (~450行)

Markdown → JSON 编译器：

| 类 | 功能 |
|---|------|
| `BrickCompiler` | 编译单个文件 |
| `BrickRegistry` | 积木注册表，管理索引 |
| `RenderMode` | 渲染模式 (FULL/COMPACT) |

关键函数：
- `compile()` — 解析 Markdown，生成 DomainBrick
- `compile_directory()` — 批量编译
- `rebuild_index()` — 重建索引
- `get_injection_bricks()` — 获取注入积木（预算控制）

#### 2.3 brick_maker.py (~550行)

积木制作主流程：

| 类 | 功能 |
|---|------|
| `BrickMakerConfig` | 配置 |
| `BrickCandidate` | 积木候选 |
| `BrickMakerResult` | 制作结果 |
| `BrickMaker` | 制作器主类 |
| `BrickReviewWorkflow` | 审核工作流 |

流程：
1. 接收多项目提取结果 (SynthesisDecision)
2. LLM 辅助知识抽取和归类
3. 生成积木候选（Markdown + YAML）
4. 人工审核后编译入库
