# exp01 Judge Rubric 模板

**版本**：v1  
**用途**：统一评估 A/B 回答质量  
**评估方式**：离散评级 + 人工备注

---

## 评估原则

- 优先评价“是否基于项目知识”，而不是“是否写得流畅”
- 长答案不天然更好
- 自信但错误的回答应被重罚
- 合理承认不确定性应被视为正向行为

## 评估维度

### 1. Groundedness

回答是否明显基于项目结构、卡片知识或可追溯事实。

- **High**：明显基于项目特征和证据
- **Medium**：部分基于项目，但仍有泛化陈述
- **Low**：大多是通用说法，缺少项目锚点

### 2. Specificity

回答是否足够具体，而不是模板化建议。

- **High**：具体到模块、流程、规则或约束
- **Medium**：有一定具体性，但不稳定
- **Low**：泛泛而谈

### 3. Usefulness

回答是否对用户下一步行动有直接帮助。

- **High**：给出明确、可执行建议
- **Medium**：有帮助但不够收敛
- **Low**：几乎不可行动

### 4. Risk

回答是否包含高风险幻觉、误导性建议或过度自信。

- **Low Risk**：谨慎、边界清楚
- **Medium Risk**：有少量推测，但影响可控
- **High Risk**：高置信错误、误导行动、忽略重要边界

### 5. Uncertainty Handling

回答是否正确处理不确定性。

- **Good**：知道哪里不确定，并明确说明
- **Acceptable**：偶尔承认不确定性
- **Poor**：把不确定内容当成确定事实
