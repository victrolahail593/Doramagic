## 2. 架构分裂决策（新增，v5 未涵盖）

### 两个独立系统

Doramagic 不是一个系统，是两个可独立运行的系统：

**系统 A：Doramagic 终端（OpenClaw Skill）**
- 完整独立的知识提取与 Skill 锻造软件
- 包含完整的 Stage 0-5 提取管线 + Phase A-H 多项目管线
- 运行在 OpenClaw 平台上
- **可以不依赖任何外部服务独立工作**
- 类比：考古学家，能独立挖掘和分析

**系统 B：预提取云端 API（可选增强服务）**
- 静态知识仓库，不做实时计算
- 两个 API 接口：
  - **查询接口**：Doramagic 终端拉取预提取的领域知识（领域图谱、知识原子、积木）
  - **提交接口**：管理员推送/更新预提取数据（批量提取结果、图谱更新，初期内部使用）
- Doramagic 连接到它时，获得 Brick 效应加成（已验证 +20% 提取质量）
- Doramagic 不连接它时，照样能完整工作
- 类比：图书馆，考古学家有它工作更好，没它也能干活
- 运行在 Mac mini (192.168.1.104)

### 关键约束
- Doramagic 终端不能与 API 强绑定
- API 只做离线预计算结果的存储和查询
- 等 Doramagic 终端发来请求才响应，不主动推送

## 3. 首批预提取领域决策（新增）

基于 Grok + Codex + Gemini 三方独立调研，确定 6 个首批预提取领域：

| # | 领域 | 范围 |
|---|------|------|
| 1 | 广义个人财务 | 记账/预算 + 投资理财 + 房产投资 + 财务知识 |
| 2 | 笔记/PKM | 块编辑/文档树 + 双向链接 + local-first + 知识回流 |
| 3 | 私有云与自托管基础设施 | 容器编排 + 反代/网络 + 认证/SSO + 监控 + 备份 |
| 4 | 健康数据（综合角度） | 健身追踪 + 异构设备数据对齐 + 协议逆向 + 去噪 |
| 5 | 信息摄取与内容管理 | RSS/过滤 + Anti-AI 去噪 + OCR/Paperless + 网页存档 |
| 6 | 智能家居与生活自动化 | 家居自动化规则 + 设备集成 + 能源监控 + 场景联动 |

**选择原则**：知识独特性 > 供需缺口 > 平台适配
**每个领域预提取 5-7 个精选项目，成本 $10-40/领域**

淘汰的领域及原因：
- 习惯追踪：知识通胀陷阱（LLM 本身处理得好，增量价值低）
- 食谱/膳食：同上
- 旅行规划：GitHub 项目池太散，冷启动覆盖不够
- 日历/任务：ClawHub 红海（126 个 Productivity skill）
- 电子遗产：需求太窄，飞轮转不起来
