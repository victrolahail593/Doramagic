## 关键决策

**为什么 tag 已存在但还是成功发布？**
ClawHub 发布时已经在 GitHub 打了 v12.4.6 tag（先于此次同步）。publish_to_github.sh 强制推送了 main 分支（force push），但 tag push 被拒绝（tag 已存在）。这不影响发布结果：代码已经是最新的，tag 指向的 commit 是 ClawHub 同步时的版本。GitHub Release 直接使用已存在的 tag 创建成功。

**为什么 bricks 在 git status 里显示 deleted 又消失了？**
package_skill.sh 先 `rm -rf skills/doramagic/bricks`，再从 `bricks/` 重新 cp。在两步之间的 git status 会显示 deleted，执行完后恢复正常状态。

## 遇到的问题

1. **tag 已存在冲突**：ClawHub 在发布时创建了 v12.4.6 tag，publish_to_github.sh 尝试再次推送时被拒绝。解决：忽略 tag 冲突，直接用已存在的 tag 创建 GitHub Release。
2. **gh release create --body 不存在**：正确参数是 `--notes`。
