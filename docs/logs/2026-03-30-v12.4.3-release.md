**为什么修复 preflight 误报而不是跳过**：
- smart_home.yaml 中的 `192.168.20.0/24` 是 IoT VLAN 最佳实践知识内容，不是内部 IP
- deploy.py 第 34 行 "password=xxx" 是注释说明屏蔽模式，不是硬编码密钥
- docs/ 有意跟踪用于内部日志，publish_to_github.sh 发布时已通过 INTERNAL_DIRS 排除
- 正确做法是更新 preflight 排除规则，而不是忽略或跳过

**SKILL.md 数字问题**：
- "7904+" 硬编码在 SKILL.md 中，知识库每次扩展就会过时
- 改为泛化描述（"从知识库中匹配..."）后，永不需要为此单独更新 SKILL.md
