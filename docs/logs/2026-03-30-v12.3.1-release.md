# v12.3.1 发布
日期: 2026-03-30
执行者: Claude Code sonnet

## 做了什么

发布 v12.3.1，主要包含三项功能修复和一项构建修复。

### 功能变更

1. **OpenClaw 配置直通**（`capability_router.py`）
   - 新增 `CapabilityRouter.from_openclaw_config()` 方法
   - 优先读取 `~/.openclaw/openclaw.json`，不再需要手动维护 `models.json`
   - `flow_controller_builders.py` 的 `build_capability_router()` 优先尝试 OpenClaw 配置，回退到 models.json

2. **LLM Adapter 修复**（`llm_adapter.py`）
   - Anthropic 响应解析修复：原来只取 `resp.content[0].text`，现在 join 所有 text block
   - 支持显式 `api_key` 字段（用于 `openclaw.json` 声明的模型）
   - Anthropic client 构建重构：支持同时传 `base_url` + `api_key`

3. **构建修复**（`pyproject.toml`）
   - `skills/` 目录加入 ruff `exclude` + `force-exclude = true`
   - `.gitignore` 新增：`web/`、`*.docx`、`*营销*.md`、skills 测试副本

### 发布过程中发现的问题

1. **pre-commit 失败**：`skills/doramagic/packages/` 没有 per-file-ignores，导致 ruff 报 E402/E501/F821。
   - 根因：`packages/` 的 per-file-ignores 没有对应 `skills/doramagic/packages/`
   - 解决：改为在 ruff `exclude` 直接排除整个 `skills/` 目录（打包产物，源码在 `packages/`）

2. **后台进程卡死**：release 脚本在后台运行时，pytest 进程 CPU 占用 0%，进度停滞
   - 根因：后台进程以 SN（低优先级）运行，pytest 被挂起
   - 解决：kill 后台进程，手动分步执行 release 流程

3. **工作树不干净**：发现 `web/`（Pencil 设计文件）、营销文档未 gitignore
   - 解决：补充 .gitignore 规则

## 关键决策

- `skills/` 整体 exclude 而非逐文件 per-file-ignores：skills 是打包产物，无需独立 lint
- 发布脚本后台进程问题没有修复脚本，而是手动分步跑——因为这是偶发问题，根因是 OS 进程调度

## 遇到的问题

无新问题。上述问题均为发布流程中的常见障碍。

## 踩坑记录

- ruff `exclude` 在显式指定文件路径时不生效，需要 `force-exclude = true`（ruff 文档不够显眼）
- 后台 bash 进程如果输出被 pipe（如 `| grep`），Claude Code 的 background task 输出文件会是空的
