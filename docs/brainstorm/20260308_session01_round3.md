## Tangsir 的两个关键洞察

### 洞察 1：透明过程 = 核心价值
- 用户必须看到 Agent 在做什么（步骤可见）
- Agent 必须解释为什么这么做（方法可见）
- 用户可以随时暂停、纠正、补充（可干预）
- 类比：像埃森哲的战略顾问在你面前工作
- 深层价值：用户在解决问题的同时顺便学到专业知识

### 洞察 2：用户输入需求，而非 GitHub 链接
- 用户不知道 GitHub 上有什么，他们只知道自己的问题
- AllInOne 负责匹配合适的项目知识
- 追问贯穿始终（边做边问），不是预先问完所有问题
- 这是真正的顾问体验

## 产品灵魂定义
> AllInOne 是一个"能干活的 AI 顾问团队"，每个顾问的专业能力来自开源世界的知识沉淀，
> 用户通过对话协作解决真实问题，全程看得见、插得上话。

## 三大产品原则
1. **问题驱动** — 用户带问题来，AllInOne 找方案
2. **过程透明** — 每步做什么、为什么，全程可见
3. **协作式执行** — 用户是搭档，不是旁观者

## UI 核心设计
- 步骤进度条（✅ 已完成 / ▶️ 进行中 / ○ 待执行）
- 方法说明卡片（解释当前使用的方法论）
- 干预控件（暂停 / 发言 / 跳过）
- 对话区（用户随时可以补充信息或纠正）

## 完整用户流程
用户描述问题 → AllInOne 理解+追问 → 匹配项目知识 → 透明可干预地执行 → 结果交付
