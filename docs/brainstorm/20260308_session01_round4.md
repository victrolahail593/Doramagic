**Date**: 2026-03-08

## Tangsir 的关键提问
GitHub 是否有用户互动？外部信息源是否对"灵魂"构成价值？

## 发现：灵魂有两半

### 代码半魂（HOW + WHAT）
- 来源：GitHub 仓库内的代码、文档、测试
- 内容：方法论、逻辑规则、数据结构、工作流程
- 覆盖知识类型 ①②③④

### 社区半魂（WHY + CONTEXT）
- 来源：GitHub Discussions、Stack Overflow、Reddit、HN、博客、YouTube、Discord
- 内容：真实用法、踩坑经验、行业常识、最佳实践、使用场景
- 覆盖知识类型 ⑤⑥

### 关键洞察
- 只有两半合在一起才是完整可用的知识
- 社区半魂把 AllInOne 的知识提取能力从 ~60% 提到 ~80%
- 知道怎么做 = 基础；知道为什么、什么时候别这么做 = 专家

## 被低估的 GitHub 功能
- **Discussions**：开放论坛，用户交流心得、提问、分享
- **Wiki**：社区维护的详细文档

## 外部信息源清单
- 技术社区：Stack Overflow, Reddit, Dev.to, 知乎/掘金
- 内容平台：博客, Medium, YouTube, 播客, 会议演讲
- 社交媒体：Twitter/X, HN, Discord/Slack, Telegram
- 聚合平台：AlternativeTo, ProductHunt, G2
- 包管理器：npm/PyPI（下载量、评分、被引用数）

## 更新后的灵魂提取流程
原五步（代码半魂）+ 新增第六步（社区半魂搜索）→ 合并为完整灵魂文档
