## 完整示例（python-dotenv）

```markdown
# python-dotenv — 灵魂

## 1. 解决什么问题？
开发者需要管理数据库密码、API Key 等配置，但不想把它们硬编码在代码里或每次手动 export。

## 2. 没有它会怎样？
每次启动终端都手动 export 十几个环境变量，或者把密钥直接写在代码里（不安全），或者用复杂的配置管理工具（太重）。

## 3. 核心承诺
让你把配置写在 .env 文件里，代码里一行 load_dotenv() 就能用，不改变你原来用 os.getenv() 的习惯。

## 4. 兑现方式
读取 .env 文件，逐行解析 KEY=VALUE，把键值对注入 os.environ。

## 5. 一句话总结
python-dotenv 让开发者可以把配置存在 .env 文件里自动加载，而不用手动 export 或把密钥硬编码到代码中。

## 6. 设计哲学
python-dotenv 的核心信念是"模拟手动 export 行为"。.env 文件就像一个你在终端里手动执行的 export 脚本，只不过是自动的、静默的。所有看似奇怪的默认行为——不覆盖已有变量、遵循 shell 引号规则、不验证值的格式——都是因为 shell 的 export 就是这样工作的。理解了这一点，你就不会再对任何默认行为感到困惑。

## 7. 心智模型
把 .env 想成一个在应用启动时静默执行的 shell 脚本。每一行就是一条 export KEY=VALUE 命令。如果你理解 shell 变量的行为（不覆盖已有变量、引号控制解析、不做格式校验），你就理解了 python-dotenv 的全部行为。
```
